⚡ AgroSmart — Pipeline de Streaming de Dados

Fluxo de dados em tempo real: sensores → Apache Kafka → processamento → dashboard

APACHE KAFKA · DOCKER COMPOSE
Etapa 1 🌾 Sensores de Campo gerador.py
  • Talhões 1–8 simulados
  • Temperatura, umidade, praga
  • 15% de leituras anômalas
  • Intervalo: 5 segundos
Etapa 2 📨 Kafka Producer producer.py
  • Publica JSON no tópico
  • Chave: talhao_id
  • Retry automático
  • Broker: kafka:29092
Etapa 3 🗃️ Apache Kafka confluentinc/cp-kafka:7.6
  • Tópico: sensor_leituras
  • Partições por talhão
  • Retenção: offset mais recente
  • Porta externa: 9092
Etapa 4 📥 Kafka Consumer main.py (MODO=consumer)
  • Consome tópico em loop
  • Persiste em CSV thread-safe
  • Group: agro-dashboard
  • Fallback → modo local
Etapa 5 ⚡ Motor de Regras regras.py
  • Infestação: folhas>30% AND praga≠nenhuma
  • Irrigação: umid<30% AND nível=baixo
  • Temperatura: >35°C OR <12°C
  • Avalia últimas 50 leituras
Etapa 6 📊 Dashboard Flask Flask · GET /api/dados
  • Alertas classificados por cor
  • Polling AJAX a cada 3s
  • Porta: 5000
  • Kafka UI: porta 8080

📦 Tópico Kafka

  • Nomesensor_leituras
  • Chave da mensagemtalhao_id (string)
  • SerializaçãoJSON UTF-8
  • Produtorproducer.py — 1 msg/5s
  • Consumidormain.py — grupo agro-dashboard
  • Retençãooffset latest (streaming puro)

📋 Schema da Mensagem JSON

{ "timestamp": "2026-06-23T14:30:05", "talhao_id": 4, "perc_folhas_doentes": 18.7, "praga_detectada": "nenhuma", "umidade_solo_pct": 62.3, "temperatura_c": 24.1, "nivel_irrigacao": "normal" }
